# Contribution Guidelines

Thank you for your interest in contributing to Awesome Orca Note! 

[English](#english) | [中文](#中文)

---

## English

### How to Contribute

Please ensure your pull request adheres to the following guidelines:

1. **Search previous suggestions** before making a new one, as yours may be a duplicate.

2. **Make an individual pull request for each suggestion.**

3. **Use the following format:**
   ```markdown
   - **[Plugin Name](link)** - Brief description of what the plugin does.
     - Author: [@username](link)
     - License: MIT
   ```

4. **New categories or improvements to the existing categorization are welcome.**

5. **Keep descriptions short and simple, but descriptive.**

6. **Start the description with a capital letter and end with a period.**

7. **Check your spelling and grammar.**

8. **Make sure your text editor is set to remove trailing whitespace.**

9. **The pull request should have a useful title and include a link to the plugin and why it should be included.**

### Quality Standards

To be added to the list, plugins should meet the following criteria:

- **Functional**: The plugin should work as described
- **Documented**: Include basic documentation or README
- **Maintained**: Shows signs of active maintenance or is stable
- **Relevant**: Must be related to Orca Note
- **Legal**: Must have an appropriate open source license or be freely available

### Adding a New Plugin

1. Fork this repository
2. Add your plugin to the appropriate category in alphabetical order
3. Ensure the description is clear and concise
4. Include author information and license
5. Test that all links work correctly
6. Submit a pull request with a clear description

### Updating Your Pull Request

Sometimes, we may ask you to edit your pull request before it is included. This is normally due to spelling errors or because your PR didn't match the guidelines.

Please use the GitHub interface to edit files in your pull request, or amend your commits and force push.

## 中文

### 如何贡献

请确保您的拉取请求遵循以下指南：

1. **在提出新建议之前搜索以前的建议**，因为您的建议可能是重复的。

2. **为每个建议单独提出一个拉取请求。**

3. **使用以下格式：**
   ```markdown
   - **[插件名称](链接)** - 插件功能的简要描述。
     - 作者：[@用户名](链接)
     - 许可证：MIT
   ```

4. **欢迎新的类别或对现有分类的改进。**

5. **保持描述简短但具有描述性。**

6. **描述以大写字母开头，以句号结尾。**

7. **检查您的拼写和语法。**

8. **确保您的文本编辑器设置为删除尾随空格。**

9. **拉取请求应该有一个有用的标题，并包含指向插件的链接以及为什么应该包含它。**

### 质量标准

要添加到列表中，插件应满足以下标准：

- **功能性**：插件应按描述工作
- **文档化**：包含基本文档或 README
- **维护性**：显示出活跃维护的迹象或处于稳定状态
- **相关性**：必须与 Orca Note 相关
- **合法性**：必须具有适当的开源许可证或可以免费使用

### 添加新插件

1. Fork 此仓库
2. 将您的插件按字母顺序添加到适当的类别
3. 确保描述清晰简洁
4. 包含作者信息和许可证
5. 测试所有链接是否正常工作
6. 提交包含清晰描述的拉取请求

### 更新您的拉取请求

有时，我们可能会要求您在包含之前编辑您的拉取请求。这通常是由于拼写错误或您的 PR 不符合指南。

请使用 GitHub 界面编辑拉取请求中的文件，或修改您的提交并强制推送。

---

Thank you for your contributions! 感谢您的贡献！
